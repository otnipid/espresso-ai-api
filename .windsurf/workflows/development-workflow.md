# Development Workflows

## Cross-Repo Change

When changing interfaces:

1. Update relevant docs
2. Add change notes
3. Commit changes
4. Inform dependent repos
5. Adapt consumers

## New Feature
1. Review vision.md
2. Create feature spec
3. Architecture review
4. Review impact
5. Implementation plan
6. Implement
7. Build
8. Test
9. Document

## Bug Fix
1. Reproduce
2. Add failing test
3. Fix
4. Refactor
